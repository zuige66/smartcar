# STM32 智能小车控制系统

## 项目概述

本项目是一个基于STM32F103C8T6微控制器的智能小车控制系统，采用FreeRTOS实时操作系统进行任务管理，集成了超声波测距、OLED显示、LED指示和串口通信等功能。

## 硬件配置

### 微控制器
- **芯片**: STM32F103C8T6
- **内核**: ARM Cortex-M3
- **主频**: 72MHz
- **Flash**: 64KB
- **RAM**: 20KB

### 外设配置
| 外设 | 功能 | 引脚配置 |
|------|------|----------|
| TIM1 | 超声波Echo信号捕获 | PA10 (CH3) |
| TIM2 | 电机PWM输出 | PA0(CH1), PA1(CH2) |
| TIM4 | 系统时基 (1ms中断) | - |
| USART2 | 串口通信 | PA2(_TX), PA3(RX) |
| GPIOA Pin4 | 左电机正转控制 (AIN1) | 推挽输出 |
| GPIOA Pin5 | 左电机反转控制 (AIN2) | 推挽输出 |
| GPIOA Pin6 | 右电机正转控制 (BIN1) | 推挽输出 |
| GPIOA Pin7 | 右电机反转控制 (BIN2) | 推挽输出 |
| GPIOA Pin11 | 超声波Trig触发 | 推挽输出 |
| GPIOA Pin12 | LED指示 | 推挽输出 |
| I2C1 | OLED显示屏 | PB6(SCL), PB7(SDA) |
| GPIOA Pin8 | 右侧循迹传感器 (X3) | 输入 |
| GPIOA Pin9 | 最右侧循迹传感器 (X4) | 输入 |
| GPIOB Pin14 | 最左侧循迹传感器 (X1) | 输入 |
| GPIOB Pin15 | 左侧循迹传感器 (X2) | 输入 |

### 超声波传感器 (HC-SR04)
- **Trig引脚**: PA11
- **Echo引脚**: PA10
- **测量范围**: 2cm - 400cm
- **测量精度**: ±3mm
- **工作原理**: 通过测量超声波往返时间来计算距离

## 软件架构

### 操作系统
- **RTOS**: FreeRTOS (CMSIS_OS2接口)
- **调度策略**: 抢占式优先级调度

### 任务列表

| 任务名称 | 优先级 | 栈大小 | 功能描述 |
|----------|--------|--------|----------|
| LedTask | Normal | 128*4 | LED闪烁控制，计数器递增 |
| UartTask | Low | 128*4 | 串口数据收发，命令解析 |
| OledTask | Low | 128*4 | OLED显示距离信息 |
| HCSR04Task | Normal | 128*4 | 超声波测距控制 |
| TrackTask | Normal+1 | 128*4 | 四路循迹传感器状态读取 |
| DriverTask | AboveNormal | 128*4 | 电机驱动控制 |
| CtrlTask | Normal | 128*4 | 综合控制逻辑，处理传感器数据并控制电机 |

### 任务间通信

#### 消息队列
| 队列名称 | 消息类型 | 队列长度 | 功能 |
|----------|----------|----------|------|
| Distance | float | 16 | 传递超声波测量的距离值 |
| LEDFlash | uint32_t | 16 | 传递LED翻转次数 |
| Track | uint8_t | 16 | 传递循迹传感器状态值 |
| DriverPWM | uint32_t | 16 | 传递电机PWM控制值 |
| MotorAction | MotorActionMsg | 16 | 传递电机控制指令和PWM值 |

#### 信号量
| 信号量名称 | 类型 | 功能 |
|------------|------|------|
| EchoBinarySem | 二进制信号量 | 超声波Echo信号完成通知，由中断回调释放，HCSR04Task获取 |

## 功能模块

### 1. 超声波测距模块 (HCSR04Task)

**功能**: 控制超声波传感器进行距离测量

**工作流程**:
1. 发送10μs的Trig触发脉冲
2. 配置TIM1通道3为输入捕获模式
3. 捕获Echo信号的上升沿和下降沿
4. 计算高电平持续时间
5. 根据声速计算距离: `distance = (falling_cnt - rising_cnt) * 0.017f` (cm)
6. 将距离值发送到Distance消息队列

**关键参数**:
- TIM1预分频: 71 (1MHz计数频率)
- TIM1周期: 65535
- 测量超时: 150ms
- 测量间隔: 100ms
- **等待方式**: 二进制信号量（优化后，减少CPU占用）

**优化说明**:
- 使用二进制信号量替代轮询等待，降低CPU占用
- 中断回调中释放信号量，任务中获取信号量
- 超时时间为150ms，超时后距离设为999.9cm

### 2. 串口通信模块 (UartTask)

**功能**: 通过USART2进行数据收发和命令解析

**通信参数**:
- 波特率: 115200
- 数据位: 8
- 停止位: 1
- 无奇偶校验

**支持的命令**:
```
led freq <value>    设置LED闪烁频率（单位：ms）
```

**示例**:
```
led freq 1000    设置LED每1000ms翻转一次
```

### 3. OLED显示模块 (OledTask)

**功能**: 在OLED屏幕上显示实时距离信息和系统状态

**显示内容**:
- 当前测量距离（单位：cm）
- LED闪烁计数
- 循迹传感器状态
- 系统状态信息

**刷新周期**: 300ms（优化后，提高稳定性）

### 4. LED指示模块 (LedTask)

**功能**: 控制LED指示灯的闪烁

**工作模式**:
- LED每500ms翻转一次（默认）
- 通过串口命令可动态调整闪烁频率
- 每次翻转时将计数器的值发送到LEDFlash消息队列

### 5. 循迹模块 (TrackTask)

**功能**: 读取四路循迹传感器状态

**硬件配置**:
| 传感器 | 引脚 | 功能 |
|--------|------|------|
| X1 | PB14 | 最左侧循迹传感器 |
| X2 | PB15 | 左侧循迹传感器 |
| X3 | PA8 | 右侧循迹传感器 |
| X4 | PA9 | 最右侧循迹传感器 |

**工作流程**:
1. 读取四个循迹传感器的状态（0=检测到黑线，1=未检测到）
2. 组合成8位状态值（X4 X3 X2 X1）
3. 通过串口发送传感器状态
4. 将状态值发送到Track消息队列

**状态值格式**:
```
X4:1 X3:0 X2:0 X1:1 | STA:0x9
```

**工作模式**:
- 每500ms读取一次传感器状态
- 状态值通过Track消息队列传递给其他任务

### 6. 电机驱动模块 (DriverTask)

**功能**: 控制左右电机的速度和方向

**硬件配置**:
| 电机 | 控制引脚 | PWM引脚 | 功能 |
|------|------------|----------|------|
| 左电机 | PA4(正, AIN1), PA5(反, AIN2) | TIM2_CH1 | 左轮驱动 |
| 右电机 | PA6(正, BIN1), PA7(反, BIN2) | TIM2_CH2 | 右轮驱动 |

**工作流程**:
1. 初始化电机控制GPIO引脚（PA4-PA7）
2. 启动TIM2的PWM输出
3. 初始化电机为停止状态
4. 从MotorAction消息队列接收控制指令
5. 根据指令设置电机方向（前进/后退）
6. 根据指令设置电机速度（PWM占空比）

**PWM配置**:
- 定时器: TIM2
- 频率: 1MHz计数频率
- PWM周期: 1000 (0-999)
- 速度范围: 0-1000

### 7. 综合控制模块 (CtrlTask)

**功能**: 处理传感器数据并控制电机动作

**工作流程**:
1. 从Distance消息队列获取距离数据
2. 从Track消息队列获取循迹传感器状态
3. 根据距离和循迹状态计算电机控制指令
4. 当距离小于10cm时，停止电机
5. 根据循迹传感器状态，控制电机转向
6. 将控制指令发送到MotorAction消息队列

**控制逻辑**:
- 距离≤10cm: 停止
- 循迹状态0000/0110/0101/1010: 前进
- 循迹状态0001/0010/0011/1101/1110: 左转
- 循迹状态0100/1000/1100/1011: 右转
- 循迹状态1111: 前进

**控制周期**: 30ms

**实现细节**:
- 使用MotorActionMsg结构体传递控制指令和PWM值
- 默认PWM速度为600（60%占空比）
- 左转/右转时PWM为400（40%占空比），实现差速转向
- 前进时PWM为550（55%占空比）
- 每30ms执行一次控制循环，确保响应及时

## 注意事项

1. **FreeRTOS API使用**: 在FreeRTOS环境下，推荐使用`osDelay()`替代`HAL_Delay()`
2. **中断优先级**: TIM1_CC_IRQn优先级设置为5，避免影响系统实时性
3. **栈空间**: 各任务栈大小为128*4字节，应避免在任务中使用大数组
4. **浮点数处理**: sprintf的%f格式符可能占用较多栈空间，建议使用整数运算替代

## 常见问题

### 1. 超声波测距超时
- 检查Trig和Echo引脚连接是否正确
- 确保传感器电源稳定（5V）
- 检查目标物体是否在测量范围内

### 2. 串口无输出
- 检查USART2引脚连接
- 确认波特率设置正确（115200）
- 检查USB转TTL模块是否正常工作

### 3. LED不闪烁
- 检查LED连接引脚（PA12）
- 确认LED限流电阻是否合适
